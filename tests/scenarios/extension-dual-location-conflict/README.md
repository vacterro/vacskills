Per RFC § 1.9's dual-location clause, the agent MUST NOT silently merge the two, guess which is newer, or pick one at random. Correct behavior: treat `.saipen/extensions/<name>/` as authoritative, the root-level copy as stale, ticket its removal (a normal `TODO`, not urgent, not `BLOCKED` -- the phase this extension attaches to still proceeds using the authoritative copy), and continue. The failure mode this guards: an agent reading the stale root copy's outdated README instructions while the real, current hook lives at the new path, silently doing the wrong thing with no signal anything is off. Behavioral test -- the assertion is which copy wins and that the conflict gets flagged, not a specific file shape.
